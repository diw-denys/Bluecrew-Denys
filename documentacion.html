Parte Individual - Panel de Administración de BlueCrew

Denys Petrychenko Oliynyk

2ºDAW

En esta primera fase de la parte individual del proyecto he optado por planificar el desarrollo del Panel de Administración de la aplicación web BlueCrew.

El objetivo de este panel es brindarle a los administradores las herramientas necesarias para gestionar la comunidad, moderar el contenido y mantener el control sobre la información de la base de datos (usuarios, eventos, ONGs, mensajes…).

A nivel técnico, esta parte del proyecto se construirá con un frontend totalmente independiente utilizando el framework Angular. Para el backend, se reutilizará la API desarrollada en grupo con Spring Boot, a la cual se le añadirán los endpoints necesarios para soportar estas nuevas funcionalidades.

Definición de las Historias de Usuario del Administrador

Para organizar y definir qué debe poder hacer el administrador en su nuevo panel, se han redactado las siguientes Historias de Usuario.

ID HU-ADM-01 Título Dashboard de Estadísticas Estimación 5 ptos
Como: administrador Quiero: ver un panel con estadísticas globales Para: conocer el estado de la aplicación rápidamente
Criterios de Aceptación
  • El panel debe mostrar tarjetas visuales con números totales de Usuarios, Eventos, ONGs y Mensajes nuevos
  • Los datos deben cargarse dinámicamente a través de los nuevos endpoints del backend
  • Esta vista debe estar protegida y ser accesible sólo por el usuarios ADMIN
ID HU-ADM-02 Título Gestión Integral de Usuarios Estimación 8 ptos
Como: administrador Quiero: tener control total sobre las cuentas de los usuarios Para: mantener la base de datos y gestionar incidencias
Criterios de Aceptación
  • Tabla con listado completo de usuarios (filtros por email, rol y estado)
  • Funcionalidad para editar datos básicos del perfil y borrar usuarios definitivamente
  • Botón de acción rápida para bloquear/desbloquear acceso
ID HU-ADM-03 Título Aprobación de ONGs Estimación 5 ptos
Como: administrador Quiero: revisar solicitudes de ONGs y gestionar sus perfiles Para: asegurar la legitimidad de las organizaciones
Criterios de Aceptación
  • Vista separada para listar ONGs en estado PENDIENTE de validación
  • Botones claros para poder Aprobar o Rechazar
  • Listado general de ONGs activas con capacidad de edición de datos o borrado (CRUD)
ID HU-ADM-04 Título Gestión de Mensajes de Contacto Estimación 8 ptos
Como: administrador Quiero: leer y responder a las consultas recibidas Para: dar soporte a los usuarios
Criterios de Aceptación
  • Listado de mensajes recibidos (nombre, email, asunto, fecha) indicando el estado
  • Vista de detalle para leer el cuerpo completo del mensaje de contacto
  • Formulario para redactar respuesta y enviar correo electrónico directamente al usuario
ID HU-ADM-05 Título Gestión Integral de Eventos Estimación 5 ptos
Como: administrador Quiero: ver y gestionar todos los eventos programados Para: moderar contenido y editar detalles si hay errores
Criterios de Aceptación
  • Vista de tabla con todos los eventos creados, indicando fecha, autor y estado
  • Funcionalidad para editar cualquier campo del evento o borrarlo definitivamente
  • Ventana de aviso pidiendo confirmación de seguridad antes de borrar un evento
ID HU-ADM-06 Título Gestión de Categorías Estimación 3 ptos
Como: administrador Quiero: crear, editar y eliminar categorías Para: mantener organizado el contenido de la web
Criterios de Aceptación
  • Listado mostrando todas las categorías actuales de la plataforma
  • Formulario rápido para añadir una nueva categoría introduciendo su nombre
  • Opciones para editar el nombre o eliminar una categoría
